# Deployment Document
## Project: Digital E-Book System ("E-Book Hub")

This document provides a comprehensive, step-by-step guide to deploying the complete E-Book System (React Frontend + FastAPI Backend) to **Render** with database integrations hosted on **Neon DB**.

No Docker containers are required. The frontend and backend are deployed as separate, optimized services to take advantage of Render's global CDN for static files and secure web services for API execution.

---

## Phase 1: Database Setup (Neon PostgreSQL)

We use **Neon DB** for our production database. Neon provides serverless PostgreSQL with auto-scaling and connection pooling.

1. **Create a Neon Project:**
   - Sign up/Log in at [Neon.tech](https://neon.tech/).
   - Click **Create Project**, name it `ebook-hub`, select your preferred database version (PostgreSQL 15 or 16), and select a region closest to your users.
   
2. **Retrieve Connection Details:**
   - In the Neon console, locate the **Connection Details** box.
   - Select **Connection string** and check the **Pooled connection** checkbox (recommended for serverless environments to handle high traffic).
   - Copy the URI. It will look like:
     ```text
     postgresql://neondb_owner:npg_xyz123@ep-some-host.us-east-2.neon.tech/neondb?sslmode=require
     ```

3. **Save Connection Details:**
   - Keep this string handy. We will input it as `DATABASE_URL` in the Backend deployment settings.

---

## Phase 2: Backend API Deployment (Render Web Service)

The backend runs the Python 3.10/3.11 FastAPI server.

1. **Log in to Render:**
   - Sign up or log in at [Render.com](https://render.com/).
   - Connect your GitHub or GitLab account containing the project repository.

2. **Create a New Web Service:**
   - Click the **New +** button on the dashboard and select **Web Service**.
   - Select your repository containing the E-Book system code.

3. **Configure Service Details:**
   - **Name:** `ebook-hub-api`
   - **Environment:** `Python`
   - **Region:** Choose the same region as your Neon DB to minimize latency.
   - **Branch:** `main` (or your active release branch)
   - **Root Directory:** `backend` *(Crucial: This tells Render to run commands inside the backend folder)*
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - **Instance Type:** `Free` (or any paid tier depending on SLA)

4. **Configure Environment Variables:**
   Click the **Advanced** button and add these variables:
   - `DATABASE_URL` = *(Your Neon DB connection string retrieved in Phase 1)*
   - `JWT_SECRET` = *(Generate a secure random key using: `openssl rand -hex 32`)*
   - `PORT` = `8000`

5. **Deploy Service:**
   - Click **Create Web Service**. Render will spin up the environment, install python libraries, establish database connections, initialize database tables, and run the FastAPI server.
   - Copy the public URL of your backend. It will look like:
     ```text
     https://ebook-hub-api.onrender.com
     ```
   - *Note: Check the Render logs. You should see `Application startup complete`.*

## Phase 3: Frontend Client Deployment (Render Static Site)

The frontend is built using Vite React and served as a high-performance static site.

1. **Create a New Static Site:**
   - Go back to the Render dashboard, click **New +**, and select **Static Site**.
   - Select the same repository.

2. **Configure Service Details:**
   - **Name:** `ebook-hub`
   - **Branch:** `main`
   - **Root Directory:** `frontend` *(Crucial: This runs build processes inside the frontend folder)*
   - **Build Command:** `npm install && npm run build`
   - **Publish Directory:** `dist` *(This is the output folder generated by Vite)*

3. **Configure Environment Variables:**
   Add this environment variable so that the compiled React application connects to your deployed backend API:
   - `VITE_API_URL` = *(The public URL of your deployed Backend Web Service, e.g., `https://ebook-hub-api.onrender.com`)*
   
   *(Note: Ensure there is no trailing slash on this URL).*

4. **Deploy Static Site:**
   - Click **Create Static Site**. Render will compile the React code, bundle stylesheets with Tailwind, and deploy the assets.
   - Once completed, you will receive a public URL for your storefront, e.g., `https://ebook-hub.onrender.com`.

## Phase 4: Database Seeding (Neon DB Production)

Once the backend is live, you must seed the Neon DB with the initial book catalog:

1. **Locate your local project directory:**
   Open a terminal and navigate to the `backend/` directory.

2. **Configure your local `.env` temporary connection:**
   Open your local `backend/.env` file and temporarily change `DATABASE_URL` to point to your **Neon DB connection string**.
   
3. **Execute the Seeder Script:**
   Run the python seeder locally. Because it uses the production connection string, it will seed your Neon DB live:
   ```bash
   python seed.py
   ```
   *(Note: After seeding is completed successfully, you can change your local `.env` back to SQLite `sqlite:///./ebook_system.db` for local testing).*

## Phase 5: Verification & Post-Deployment Checklist

1. **Verify API is Healthy:**
   - Open a browser and visit `https://YOUR-BACKEND-URL.onrender.com/` (it should return a healthy status JSON).
   - Go to `https://YOUR-BACKEND-URL.onrender.com/docs` to see the interactive Swagger UI.

2. **Test Storefront Navigation:**
   - Visit `https://YOUR-FRONTEND-URL.onrender.com`.
   - Ensure the catalog loads the 6 seeded books.
   - Click **Login** and sign in using:
     - **Email:** `admin@ebook.com`
     - **Password:** `admin123`
   - Navigate to `/admin` to verify that database CRUD permissions work.
   - Log out, sign in as `user@ebook.com` / `user123`, add items to your cart, proceed to checkout, make a purchase, and download a book PDF to verify the secure delivery pipeline.

## Monitoring & Rollback Strategies

- **Real-Time Logs:** View runtime warnings and request logs on the **Logs** tab of your Render Web Service.
- **Rollback Deploys:** If a commit breaks the production build, open the **Deploys** tab of the service on Render, locate the previous successful deployment, and click **Rollback** to instantly restore service.
